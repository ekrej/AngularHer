Keuze voor NoSQL

de keuze voor NoSQL is gemaakt omdat NoSQL werkt met JSON. JSON is in vergelijking met SQL storage een stuk voordeliger in opslag. Een nadeel van SQL is dat relaties moeilijker te maken zijn, maar aangezien deze web applicatie met simpele objecten met weinig, en alleen simpele relaties, werkt en is dit geen groot ding. Ook kan JSON objecten in objecten opslaan wat voor kleine objecten geen probleem is. zoals de comments in een discussion, daar stop ik een referntie in naar de comment en die worden bij het ophalen van de discussion opgehaald.
